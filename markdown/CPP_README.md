# `graph_prep` — Complete Technical Reference

> This document is written as a **zero-knowledge reference**: everything you need to build a Python wrapper is here. Assumptions about the C++ internals are explicitly stated. Silent failure modes are called out with ⚠️.

---

## 1. Compilation

```bash
mkdir -p bin
g++ -o bin/graph_prep main.cpp param.cpp -O3 -std=c++11
```

The binary entry point is `main.cpp`. All other `.cpp` files are `#include`d transitively — do **not** compile them separately.

---

## 2. Global Runtime Parameters

These are global variables set at the start of every run. Their defaults and override mechanisms matter for reproducibility.

| Variable | Default | Type | Description |
|----------|---------|------|-------------|
| `K` | `32` | `unsigned int` | MinHash sketch width (number of min-hash values per layer) |
| `L` | `1` | `unsigned int` | Number of independent sketch layers / samples |
| `SEED` | `42` | `unsigned int` | Global RNG seed used by all `std::mt19937` instances |

**How they get set at runtime:**

- For `sketch`: `K` = argv[5] (required), `L` = argv[6] (optional), `SEED` = argv[7] (optional).
- For all **experiment tasks** (`ExactD`, `GloD`, `PerD`, etc.): `SEED` is set from argv[5] **first**, then `K` is also set from argv[5] for any task that needs K. This means **argv[5] sets both `SEED` and `K` simultaneously** for experiment tasks. There is no way to set them independently.

## 3. Required Dataset Directory Structure

Every experiment task reads files from a hardcoded directory layout. **The dataset name IS the directory name** — these are not configurable.

```
<working_directory>/
├── bin/
│   └── graph_prep
│
├── <DATASET>/                          ← Dataset input folder (e.g. "HGBn-ACM")
│   ├── meta.dat                        ← Single line: "Vertices number: <N>"
│   ├── node.dat                        ← TSV: nodeID \t ... \t type1,type2,...
│   ├── link.dat                        ← TSV: srcID \t dstID \t edgeType1,edgeType2,...
│   ├── <DATASET>-cod-global-rules.dat  ← Master rule file (stack-machine format, single line)
│   ├── <DATASET>-cod-global-rules.limit ← Variant rule file (used by mpcount)
│   ├── <DATASET>-cod-global-rules.dat1 ← Generated by lensplit (length-1 rules)
│   ├── <DATASET>-cod-global-rules.dat2 ← Generated by lensplit (length-2 rules)
│   ├── <DATASET>-cod-global-rules.dat3 ← Generated by lensplit (length-3 rules)
│   ├── qnodes_<DATASET>.dat            ← One integer node ID per line (used by hg_stats)
│   └── cod-rules_<DATASET>.limit       ← Per-query rule file (used by hg_stats)
│
└── global_res/                         ← Ground truth output folder (CREATE MANUALLY)
    └── <DATASET>/
        ├── df1/
        │   ├── hg_global_r<TOPR>.res           ← Written by: ExactD+ stdout redirect
        │   └── hg_global_greater_r<TOPR>.res   ← Written by: ExactD  stdout redirect
        └── hf1/
            ├── hg_global_r<TOPR>.res           ← Written by: ExactH+ stdout redirect
            └── hg_global_greater_r<TOPR>.res   ← Written by: ExactH  stdout redirect
```

⚠️ **The binary never creates the `global_res/` directory tree.** You must create it manually before running ExactD/ExactH or any Glo*/Per* task.

⚠️ **`<TOPR>` in filenames is the raw string passed as argv[3]**, e.g. `"0.1"` → file is `hg_global_r0.1.res`. The file lookup is a direct string concatenation — `0.10` and `0.1` are different filenames.

## 4. Rule File Format

All experiment tasks read a **single-line** rule file using a stack-machine encoding. Understanding this is essential to avoid generating empty output.

### 4.1 Opcodes

| Integer | Meaning |
|---------|---------|
| `-1` | Set state = VARIABLE. The next positive integer is the **final edge type** of a variable rule. The rule is processed immediately after. |
| `-2` | Push direction `FORWARD` (→) for the next edge |
| `-3` | Push direction `BACKWARD` (←) for the next edge |
| `-4` | **Pop**: remove the last pushed edge type, direction, and node type from the current path stack |
| `-5` | Set state = INSTANCE. The next positive integer is the **instance node ID**. The rule is processed immediately after. |
| `≥ 0` | An edge type integer. Behavior depends on current state: if state is unset (`-1`), it is pushed as an intermediate edge; if state is VARIABLE, it is pushed as the final edge and the rule fires; if state is INSTANCE, it is used as the instance node ID and the rule fires. |

### 4.2 How a rule is built: example

Meta-path: node `→[type 0]→ node →[type 2]→ node` as a **variable** rule:

```
-2 0 -2 -1 2 -4 -4
```

Step by step:
1. `-2` → push direction FORWARD
2. `0` → push edge type 0 (state is unset, so intermediate)
3. `-2` → push direction FORWARD
4. `-1` → set state = VARIABLE
5. `2` → push edge type 2 AND fire the rule (pattern is now `[0,2]` both forward)
6. `-4` → pop type 2 / direction
7. `-4` → pop type 0 / direction

### 4.3 Multiple rules in one file

Rules are encoded as a tree traversal. A file might contain many rules sharing a common prefix:

```
-2 0 -2 -1 2 -4 -3 -1 4 -4 -4
```
This encodes TWO variable rules: `[0→, 2→]` and `[0→, 4←]`.

### 4.4 Instance vs Variable rules

- **Variable rule**: no fixed starting node. `Peers()` is called with `instance == -1`, so it searches from all nodes. Triggered by `-1`.
- **Instance rule**: anchored to a specific node. `Peers()` is called with `instance = <node_id>`. Triggered by `-5 <node_id>`.

### 4.5 `materialize` and `sketch` rule files

These two tasks use `parse_first_rule_from_file()`, which parses **only the first complete rule** in the file and then stops. Everything after the first rule is ignored.

## 5. Command Reference

### 5.1 `materialize`

**Purpose:** Exact BFS-based materialization. Finds all metapath neighbors for every node and writes the adjacency list.

```
./bin/graph_prep materialize <dataset_dir> <rule_file> <output_file>
```

| Arg | Required | Description |
|-----|----------|-------------|
| argv[2] | ✓ | Dataset directory (must contain `meta.dat`, `node.dat`, `link.dat`) |
| argv[3] | ✓ | Path to rule file (only first rule is parsed) |
| argv[4] | ✓ | Output file path |

**Output file format:**
```
<node_id> <neighbor_1> <neighbor_2> ... <neighbor_N>
```
One line per node. Only nodes that are `ractive` (active peer nodes) **and have at least one neighbor** are written. Isolated peer nodes produce no output line.

**Stdout:** Pattern debug info from `qp->print()`.

**Silent failure modes:**
- ⚠️ If `rule_file` contains an empty or malformed rule, `parse_first_rule_from_file` throws `std::runtime_error("Failed to parse any rule from file.")`.
- ⚠️ If `output_file` path's parent directory doesn't exist, the `ofstream` open fails and throws.
- ⚠️ If `peer_size == 0`, the output file is created but empty with no error.

---

### 5.2 `sketch`

**Purpose:** Approximate metapath neighbor sampling using MinHash sketches. Produces `L` separate output files.

```
./bin/graph_prep sketch <dataset_dir> <rule_file> <output_base_path> <K> [L] [SEED]
```

| Arg | Required | Default | Description |
|-----|----------|---------|-------------|
| argv[2] | ✓ | — | Dataset directory |
| argv[3] | ✓ | — | Rule file path (only first rule is parsed) |
| argv[4] | ✓ | — | Base output path (e.g., `out/graph.txt`) |
| argv[5] | ✓ | — | `K`: sketch width |
| argv[6] | ✗ | `1` | `L`: number of sketch layers / output files |
| argv[7] | ✗ | `42` | `SEED`: RNG seed |

**Output files:** The tool does **NOT** write to argv[4] directly. It writes to `L` files, inserting an index before the extension:

| Input path | L | Actual output files |
|------------|---|---------------------|
| `out/graph.txt` | 3 | `out/graph_0.txt`, `out/graph_1.txt`, `out/graph_2.txt` |
| `out/graph` | 2 | `out/graph_0`, `out/graph_1` |

**Output file format (per layer):**
```
<peer_node_id> <sampled_neighbor_1> <sampled_neighbor_2> ...
```
Each sampled neighbor list has at most `K` entries. Peers with no sketch entries produce no output line.

**Stdout:** `Materializing Sketch Layer <l> to <filename>...` for each layer.

**Silent failure modes:**
- ⚠️ If `K` or `L` is not specified as a number, `std::stoul` throws.
- ⚠️ If the output directory doesn't exist, each per-layer file silently fails to open (the code prints a warning to `cerr` but continues). Output files will not be created.
- ⚠️ If no peers are found (empty matching subgraph), all output files are created but empty.
- ⚠️ `K` passed here sets the global `K` variable which also affects sketch size. `L` sets global `L`.

---

### 5.3 `debug_rule`

**Purpose:** Parse a rule file and print the detected edge type sequence and direction sequence.

```
./bin/graph_prep debug_rule <rule_file>
```

**Stdout:**
```

DEBUG_PARSED_SEQUENCE: <et1> <et2> ...
```

Directions: `1` = forward, `-1` = backward.

- ⚠️ If no valid rule is found, throws `std::runtime_error("Failed to parse any rule from file.")`.

### 5.4 `dump`

**Purpose:** Dump the entire graph edge list to a plain text file for inspection.


**Output file format:**

Tab-separated. One line per (src, dst, edge_type) triple. Nodes with no outgoing edges produce no lines.

### 5.5 `ExactD`

**Purpose:** Exact degree-based top-R computation (strictly-greater set). Writes parseable output to stdout which must be redirected to a `.res` file to be used by `GloD`.

```
./bin/graph_prep ExactD <dataset> [topr] [beta_unused] [K_and_SEED]
```

| Arg | Default | Description |
|-----|---------|-------------|
| argv[2] | — | Dataset name (= directory name) |
| argv[3] | `"0.1"` | Top-R threshold as decimal string |
| argv[4] | `"0.1"` | Unused for this task but consumed |
| argv[5] | N/A | Sets both `SEED` and `K` to this value (see §2) |

**Input files read:**
- `<dataset>/meta.dat`, `<dataset>/node.dat`, `<dataset>/link.dat`
- `<dataset>/<dataset>-cod-global-rules.dat`

**Stdout (must redirect to file):**

For each processed rule, for each meta-layer `l`:
```
% <edge_sequence>    instance:<value>
# l=<meta_layer>
( <node_id1> <node_id2> ...          ← if strictly-greater set is < 50% of peers
) <node_id1> <node_id2> ...          ← if strictly-greater set is ≥ 50% of peers (complement encoding)
time:<seconds>
```

After all rules:
```
~time_per_rule:<seconds> s
rule_count:<count>
```

**Encoding of `(` vs `)` lines:**
- `(` = the listed nodes ARE in the strictly-greater-than set (dom set)
- `)` = the listed nodes are NOT in the strictly-greater-than set (complement; unlisted peers that are active are in the set)

⚠️ This encoding must be decoded correctly when reading the `.res` file. The `Effective_prop_opt_global_cross` reader handles this via `hg_isdoms_greater`.

**Usage:**
```bash
./bin/graph_prep ExactD HGBn-ACM 0.1 0 > global_res/HGBn-ACM/df1/hg_global_greater_r0.1.res
```

### 5.9 `GloD`

**Purpose:** Approximate Global Degree top-R using MinHash sketches. Computes F1 score against ExactD/ExactD+ ground truth files.

```
./bin/graph_prep GloD <dataset> [topr] [beta_unused] [K_and_SEED]
```

| Arg | Default | Description |
|-----|---------|-------------|
| argv[2] | — | Dataset name |
| argv[3] | `"0.1"` | Top-R threshold |
| argv[4] | `"0.1"` | Unused |
| argv[5] | N/A | Sets **both** `K` and `SEED` to this integer value |

**Input files required (must pre-exist):**
- `global_res/<dataset>/df1/hg_global_r<topr>.res` ← generated by `ExactD+`
- `global_res/<dataset>/df1/hg_global_greater_r<topr>.res` ← generated by `ExactD`
- `<dataset>/<dataset>-cod-global-rules.dat`

⚠️ If either `.res` file is missing or empty, `getline` silently reads nothing. The F1 score will be computed against empty ground truth, producing misleading/invalid results with **no error message**.

**Stdout per rule (interleaved):**
```
% <edge_sequence>    instance:<value>
# l=<meta_layer>
goodness:<f1_score_for_this_rule>
time:<seconds_for_this_rule> s
SCATTER_DATA: <rule_index>,<active_edge_count>,<time_in_seconds>
```

**Stdout summary (end of run):**
```
~goodness:<average_f1>
~time_per_rule:<seconds> s
rule_count:<count>
```

**Key stdout tokens for parsing:**
| Token | Value |
|-------|-------|
| `~goodness:` | Average F1 score across all rules |
| `~time_per_rule:` | Average wall-clock time per rule (seconds) |
| `SCATTER_DATA:` | Per-rule CSV: `rule_idx,active_edges,time_s` |
| `goodness:` | Per-rule F1 (without tilde) |

## 6. Mandatory Execution Order for Experiments

The `Glo*` and `Per*` tasks crash silently on empty `.res` files. The following order is required per dataset per `topr` value:

```bash
DATASET="HGBn-ACM"
TOPR="0.1"

mkdir -p global_res/${DATASET}/df1
mkdir -p global_res/${DATASET}/hf1

# Step 1: Generate all four ground truth files
./bin/graph_prep ExactD+  ${DATASET} ${TOPR} 0 > global_res/${DATASET}/df1/hg_global_r${TOPR}.res
./bin/graph_prep ExactD   ${DATASET} ${TOPR} 0 > global_res/${DATASET}/df1/hg_global_greater_r${TOPR}.res
./bin/graph_prep ExactH+  ${DATASET} ${TOPR} 0 > global_res/${DATASET}/hf1/hg_global_r${TOPR}.res
./bin/graph_prep ExactH   ${DATASET} ${TOPR} 0 > global_res/${DATASET}/hf1/hg_global_greater_r${TOPR}.res

# Step 2: Run approximate methods
./bin/graph_prep GloD  ${DATASET} ${TOPR} 0 32  # K=32, SEED=32
./bin/graph_prep GloH  ${DATASET} ${TOPR} 0 4   # K=4,  SEED=4
./bin/graph_prep PerD  ${DATASET} ${TOPR} 0 32
./bin/graph_prep PerH  ${DATASET} ${TOPR} 0 4
./bin/graph_prep PerD+ ${DATASET} ${TOPR} 0.1 32
./bin/graph_prep PerH+ ${DATASET} ${TOPR} 0.1 4
```

## 7. Ground Truth File Format (`.res`)

The `.res` files (produced by redirecting ExactD/ExactH stdout) are parsed by `Glo*`/`Per*` tasks. The parser only reads lines starting with `(` or `)`.

```
( 12 45 67 88          ← "dom set": listed nodes are IN the top-R set
) 3 7 22               ← "complement set": listed nodes are NOT in the top-R set
```

**Parsing logic:**
- `(` line → `isdom = true` → the listed node IDs are the set members
- `)` line → `isdom = false` → the listed node IDs are the COMPLEMENT; active nodes NOT listed are the actual set members

Lines starting with `%`, `#`, `time:`, `~`, etc. are silently ignored.

## 9. Complete Argument Position Table

All experiment tasks share this positional argument scheme:

```
argv[0]  argv[1]   argv[2]    argv[3]   argv[4]   argv[5]
binary   <TASK>    <DATASET>  [TOPR]    [BETA]    [K_AND_SEED]
```

- `TOPR`: float string, default `"0.1"`. Used in filenames literally (e.g. `"0.05"` → `r0.05.res`).
- `BETA`: float string, default `"0.1"`. Used as pruning tolerance in `PerD+`/`PerH+`. Ignored by all other tasks.
- `K_AND_SEED`: single integer that **simultaneously** sets both `K` (sketch width) and `SEED` (RNG seed).

For `sketch` only:

```
argv[0]  argv[1]   argv[2]       argv[3]      argv[4]        argv[5]  argv[6]  argv[7]
binary   sketch    <DATASET>     <RULE_FILE>  <OUTPUT_BASE>  <K>      [L]      [SEED]
```

Here `K`, `L`, and `SEED` are set independently.

## 11. Known Bugs and Gotchas

1. **`argv[5]` is used as both `K` and `SEED` simultaneously** in all experiment tasks. There is no way to independently control them. If you need reproducibility, pick a `K` value and accept that `SEED = K`.

2. ~~**`PerD` reads H-index ground truth files**~~ **FIXED.** The guard condition in `Effective_prop_opt_personalized_cross` now includes `method == "d"`, so `PerD` correctly reads from `df1/` files alongside `PerD+`.

3. **`union` (`hidden_graph_deg_union`) may segfault** due to accessing `image_union[image_union.size()-1]` when `image_union` is empty. Wrap calls to `union` in a subprocess with a timeout.

4. **Silent empty output**: If `peer_size <= 2` for every rule, `rule_count` will be 0 and `avg_time /= 0` produces NaN/inf in the summary. Check `rule_count:0` in output to detect this.

5. **`topr` string is used verbatim in filenames**. `"0.10"` ≠ `"0.1"`. Always pass the exact same string you used during ground truth generation.

6. **`lensplit` silently drops rules with 4+ edges.** No warning is emitted.

7. **`mpcount` reads `.limit` file, not `.dat` file.** The file must be named `<DATASET>-cod-global-rules.limit`.

8. **`hg_stats` reads `cod-rules_<DATASET>.limit`** (different naming convention from all other tasks — underscore before dataset name, no double-name prefix).

9. **`LOOP = 5` is hardcoded** in `param.h`. The F1 computation in `GloD`/`GloH` averages over 5 identical repetitions of each rule internally. You cannot change this without recompiling.

10. **`QNUM = 10` is hardcoded.** Personalized tasks always sample exactly 10 query nodes per rule.

11. **`PATHCOUNT = 10` is hardcoded.** Scale tasks stop after processing 10 valid rules, regardless of how many rules are in the file.
